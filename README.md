## 概要
&emsp;日々の学習内容を「記録」するだけでなく、**「後から検索しやすく、自分の成長を視覚的に実感できる」**ことをコンセプトにしたシングルページアプリケーション（SPA）です。   
&emsp;日記上にすることで、いつ学んだかを具体的にし頑張った日々を思い出すことがモチベーション維持に繋がれば良いと考えた。

## 開発の背景
&emsp;フロントエンドエンジニアを目指す過程で、学習の広さと自分の進捗具合が見えにくくなることに課題を感じました。   
&emsp;開発の過程でフロントエンドの知識を培うこともできれば良いと思います。

## 技術的挑戦と解決
&emsp;開発中に直面した問題を記載していく

### `<a>`タグの罠(2026/2/21)
&emsp;**Issue#1/Headerの作成**中、やり方を調べていくうちに`<a>`タグはページ遷移する度に画面が真っ白になる状態が発生することが判明した。[(参考文献)](https://qiita.com/naoki-atjp/items/bc8a7028aba7d3725832)<br>これを解決するためにreact-router-domをしようし、`<a>`タグから`<Link>`に切り替えることでページ遷移が

### バージョン管理(2026/2/21)

&emsp;[`<a>`タグの罠](#aタグの罠221)にて、react-router-dom によるページ遷移の実装も検討しましたが、Issue#1 の作業中に新しいライブラリを混ぜると、バージョン管理が複雑になり、混乱を招く可能性があると考えました。
そこで、まずは Issue#1 を予定通り `<a>` タグで確実に完了・マージさせ、土台を固めることを優先しました。npm install を含む本格的なルーティングの実装は、次回のIssueとして切り分けることで、作業の粒度を細かく保つ工夫をしています。

### Github認証エラー(2026/2/22)
- **問題** <br>  &emsp;githubと連携完了後に、ターミナルにて'git push'時にユーザ名とパスワードを入力するように求められた。Githubのアカウントでログインしてもできなかった。  

- **解決** <br>  &emsp;アクセストークンを発行し、
```bash
git remote set-url origin https://[アクセストークン]@github.com
```
で登録したところ解決した

### Homeはいづこへ(2026/2/23)
- **問題** <br>  &emsp;`<Link to>`に切り替えたときにHomeを見失った。
- **原因** <br>  &emsp;width:100をheaderに入れていた事により画面外にHomeがあることがコンソール画面で確認できた。
- **解決** <br>  &emsp;justifyContent: "flex-start"に変更することでheaderの隣にHomeに戻るリンクボタンを作成できた。

### 設計の見直し(2026/2/24)
- **問題と課題** <br>  &emsp;ハンバーガーボタンの実装を進める中で、「ヘッダー（Header）」と「ボタン」の重なりが発生。当初の想定ではヘッダー内にボタンを配置する予定でしたが、開発を進めるうちに、この構成ではサイドバーとの連動が難しくなることがわかった。
- **今後の方針** <br> &emsp;一旦作業中ではありボタンの追加は完了したが、ボタン追加をせずレイアウト構成から検討をし直しサイドバーとメイン機能の二つを追加する新たなIssueを作ることに決めた。

### `<div>`多すぎんねん(2026/2/25)
- **問題** <br> &emsp;Issue#8：メイン画面とサイドバーの作成において、レイアウト用の <div> が何重にも重なり、コードの可読性が低下する。
- **解決** <br> &emsp;App.tsx をレイアウトの**枠組み**に特化させ、中身（Sidebar, Header等）を個別の .tsx ファイルへ切り出す。
